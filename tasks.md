# PCCA — Refactoring Backlog

[scenarios.md](./scenarios.md) is the product-level source of truth for user/stakeholder intent. This backlog tracks known gaps between the current implementation, that product intent, and the design in [architecture.md](./architecture.md). Each task has a priority, a concrete problem, a sanctioned solution, acceptance criteria, and the impact of fixing it.

**Rules:**
- P0 tasks block daily-use dogfood. Do not start P1+ work while P0 items are open.
- New features land only after the related P0 tasks are closed.
- Every non-trivial PR should reference an existing task ID. If no task exists, add one here first.

**Priority semantics:**
- **P0** — critical correctness / reliability; user-visible silent failure today.
- **P1** — required for 14-day-use stability; spec-mandated reliability controls.
- **P2** — quality of outcomes; makes the curation actually learn / explain.
- **P3** — polish, consistency, observability, nice-to-haves.

**Recently closed (see git history for implementation):** T-01, T-02, T-04, T-05, T-06, T-13, T-22, T-24, T-28, T-30, T-31, T-34, T-36, T-37B, T-03, T-44, T-45, T-46, T-48.
